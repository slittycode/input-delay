Gamepad Polling-Rate Tester

Native macOS path: the browser's W3C Gamepad API is backed by Apple's GameController framework — the same layer that blocks terminal/SDL tools. This page derives polling rate and report-interval jitter from the deltas between distinct gamepad.timestamp values. Open in Chrome or Safari.

Detected controller: none — press any button to wake it

How to run:

  1. Press any button on the controller so the browser exposes it.
  2. Click Start.
  3. Rotate the LEFT stick in continuous circles at the outer edge for ~15 s (keeps a fresh report every poll, like gamepadla).
  4. Click Stop and read the results. A copyable JSON block appears at the bottom.

Important limitation (read this)

Browser gamepad sampling can be bounded by the page's loop rate. This page samples with a tight MessageChannel loop (faster than requestAnimationFrame) and reports the actual sampler rate above. If the sampler rate is well above the measured polling rate, the number is trustworthy. If they are close, the polling rate is a lower bound capped by the browser, not the controller's true report rate. Compare against the terminal tools' theoretical max (125/250/500/1000 Hz buckets) accordingly.
